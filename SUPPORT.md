# Support

## How to file issues and get help

This project uses GitHub Issues to track bugs, feature requests, usage questions, and reproducibility discussions.

Before opening a new issue, please search existing issues first. When filing a report, include enough detail to reproduce the problem, including environment information, commands, expected behavior, and observed behavior.

For security-sensitive reports, do not open a public GitHub issue. Follow the process in [SECURITY.md](SECURITY.md).

## Microsoft support policy

Support for this project is limited to the resources listed above.
